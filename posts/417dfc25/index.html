网络相关(2):代理及其工具使用

发表于 2026-04-17 分类于 Tools 阅读次数： Waline： 本文字数： 1.1k 阅读时长 ≈ 1 分钟

此处记录一下：网络代理工具Clash的使用

工具配置步骤举例

此前经典的Clash已删库跑路不再维护，不稳定且代理质量受影响

Fclash：基于 ClashMeta 的多平台代理客户端[Android、Windows、macOS 和 Linux]，简单易用，开源无广告。下载/更新链接，https://github.com/chen08209/FlClash/releases/

Windows配置举例

  1. 下载对应客户端的压缩包文件，解压到本地文件夹（建议非C盘）
  2. 运行工具 FlClash.exe，添加URL订阅链接，即完成导入及更新
  3. 打开代理——切换选择节点（优先实验节点）
    1. 初次配置遵循上述步骤，更新只需更改订阅或刷新即可（刷新前关闭代理服务）
    2. 手机版及其他终端类似，仅界面差异。
    3. 若个别连接失败，可能——电脑网络代理设置/本地网络延迟过高
  4. 服务失效或刷新会及时发布，如无则说明正常使用中。

IOS_Shadowrocket

预先获取 AppStore 小火箭下载共享账户

！共享账户仅限 AppStore 使用，禁止登录 iCloud 账户，否则将导致数据丢失或设备锁定。

步骤：

  1. 登录 AppStore ，下载 Shadowrocket
  2. 复制订阅链接（Subscribe）、导入订阅/更新配置
  3. 打开并选择节点、切换模式

代理/作用简介

  1. 代理的原理：本地网络受IP地址限制无法访问 Google/Github 先将访问请求发送给一个有权访问的代理服务器(如在香港)，它接收请求访问网页，再将请求“结果”返回给本地，充当了一个“代发”的桥梁作用（类比代购）——绕过网络限制、隐藏真实IP地址。
  2. 访问延迟：服务器访问速度通常很快且只获取必要数据包，所以响应时间基本可忽略，延迟一般接近本地网速。（网站通常是Https，代理通常走的是Socks协议，这类协议只转发数据包，省略其他各种信息，因此更加快捷隐私，端口默认为7890）
  3. 代理工具如Clash/Shadowrocket：图形化或命令行的代理客户端，将协议及代理服务器等整合为易用的流量调度、代理管理及网络切换服务。
    1. 代理工具必须依赖已有的代理服务器（节点）才能工作，Clash本身不提供代理服务。
    2. 使用步骤：购买了一个代理服务器（机场/订阅）——>安装clash——>导入订阅——>开启代理
      • 订阅链接：包含了所有节点的信息和一套基础分流规则
      • 出站/代理模式：
        • Rule = 智能分流（推荐，日常工作使用），根据规则自动处理国内国外流量
        • Global = 全部走代理（特殊场景临时开启）
        • Direct = 全部直连（彻底关闭代理效果）
